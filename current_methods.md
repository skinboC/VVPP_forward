# 敲击声/模态振动频谱预测模型：原理与实现文档

本文档详细介绍了本项目（基于 `src/pipeline.py`）的整体数据流向、核心组件设计，以及针对敲击声模态特征设计的多种频谱预测方案的原理。

## 1. 系统整体架构与数据流 (Pipeline Overview)

本系统旨在根据物体的三维几何结构和特定的敲击点，预测该次敲击产生的声音频谱（能量包络）。整体流程在 `MyPipeline.forward` 中实现：

1.  **输入解析与特征提取：**
    *   **几何输入**：使用八叉树（Octree）表示物体的 3D 结构，通过 `ocnn_unet.UNet` 提取每个节点的深层几何特征。
    *   **上下文特征（Global Context）**：从物体的 GNN 顶点中随机或均匀采样固定数量（如 512）的全局点，计算这些点几何特征的均值和最大值，形成代表物体整体形状的全局上下文向量 `global_features`。
    *   **局部特征（Local Feature）**：提取具体敲击点（Impact Points）的局部几何特征 `local_features`。
2.  **特征融合与声学场预测 (AcousticFieldHead)：**
    *   **位置编码**：将敲击点的 `(x, y, z)` 坐标进行正余弦高频映射（Positional Encoding），以增强对高频细节的敏感度。
    *   **交叉注意力融合 (Cross-Attention)**：局部特征与全局特征通过 MultiheadAttention 进行融合。局部特征作为 Query，全局和局部特征拼接后作为 Key/Value，使得网络能够理解“在这个特定位置敲击时，整个物体的共振响应”。
    *   **参数化预测与渲染**：融合后的特征被送入不同的预测头（Predictor），输出代表声音模态的物理参数，最终渲染（Render）为连续的 1D 频谱包络。
3.  **目标构建与 Loss 计算：**
    *   **目标提取**：从真实的梅尔频谱图（Mel-spectrogram）中，沿着时间维度取最大值，提取出真实的 1D 能量轮廓。利用 `F.adaptive_avg_pool1d` 降采样对齐到固定的 `output_dim`。
    *   **Loss 监督**：计算预测频谱与真实频谱之间的误差，包括波形回归损失、能量分布损失等。

---

## 2. 频谱预测方案详解 (Prediction Modes)

针对敲击声的物理特性（由一系列离散、极窄、数量不定的模态频率组成），我们设计了多种预测方案，可通过 `cfg.PREDICTION_MODE` 进行切换。

### 方案 0：直接回归 (Direct Prediction)
*   **模式名称**：`"direct"`
*   **原理**：最简单粗暴的方法。网络最后通过一个全连接层，直接输出一个长度为 `output_dim`（如 256）的向量，代表频谱上每个频率点的值。
*   **优缺点**：
    *   *优点*：无需任何假设，自由度最高。
    *   *缺点*：参数效率极低，没有利用到“声音由离散模态组成”的物理先验，容易过拟合到噪声或产生不平滑的伪影。

### 方案 1：二分图匹配集合预测 (Bipartite / Hungarian Matching)
*   **模式名称**：`"bipartite"`
*   **原理**：网络输出固定数量（如 12 个）的波峰（Peaks），每个波峰由 `(频率 freq, 振幅 amp, 宽度 width)` 三个参数构成。渲染时，将这 12 个高斯函数相加合成连续频谱。
*   **原意与局限**：原本设计是希望通过匈牙利算法，将预测的波峰与 Ground Truth 的真实波峰进行一对一匹配。但由于实际输入只有连续的 GT 频谱包络（难以提取无误差的离散伪标签），该方案目前退化为直接在连续域计算 Loss，类似于一种无约束的高斯混合模型（GMM）拟合。

### 方案 2：锚点分箱参数预测 (Anchor-based Prediction)
*   **模式名称**：`"anchor"`
*   **原理**：借鉴目标检测中的 Anchor 思想。将整个频率轴均匀切分为 `num_peaks`（如 12）个区间（Bins）。网络为每个区间预测 4 个值：`(基线 base_val, 相对偏移 offset, 振幅 amp, 宽度 width)`。
    *   `绝对频率 = 区间中心 + offset * 区间宽度`
*   **优缺点**：
    *   *优点*：强制网络在整个频率范围内均匀寻找波峰，避免了多个波峰扎堆在低频或高频区域，训练非常稳定。引入的 `base_val` 也能很好地拟合宽带噪声。
    *   *缺点*：敲击声的模态在频域分布极不均匀，强行划分区间可能导致密集的模态无法被分辨（一个 Bin 只能出一个模态）。同时，敲击声模态极窄，预测 `width` 是冗余的。

### 方案 3：模态锚点与可微集合预测 (Modal Anchor / Continuous Set Prediction)
*   **模式名称**：`"modal_anchor"`
*   **原理**：这是专门针对“模态合成（Modal Synthesis）”设计的物理增强方案。由于敲击声的模态是极窄的离散频率，且数量不定，该方案摒弃了宽度的预测，引入了**存在概率 (Probability)** 和**稀疏惩罚 (Sparsity Penalty)**。

该方案包含两个子模式（由配置项 `cfg.USE_MODAL_BINS` 控制）：

#### 3.1 强行划分区间模式 (Fixed Bins)
*   **配置**：`cfg.USE_MODAL_BINS = True`
*   **实现**：将频率轴划分为较细的 64 个区间。在每个区间内，网络预测 `(prob, offset, amp)`。
    *   `prob` (0~1)：表示该频率区间内**是否存在**激发的模态。
    *   `offset`：区间内的精细频率微调。
    *   `amp`：模态的振幅能量。
*   **渲染**：最终在这个区间上的有效振幅为 `prob * amp`。渲染时采用一个极小且固定的宽度（如 1 个像素宽），以保持梯度的平滑传递。

#### 3.2 无匹配集合预测模式 (Free Continuous Set Prediction) - ✨ 最推荐
*   **配置**：`cfg.USE_MODAL_BINS = False`
*   **实现**：彻底打破区间的限制。网络直接输出 64 个**自由的模态集合**。每个模态依然预测 `(prob, freq, amp)`，但此时的 `freq` 是在 `[0, 1]` 范围内的**全局绝对频率**。
*   **优势与隐式匹配**：
    *   这 64 个模态可以自由游走，完美适应某些频段模态密集、某些频段空缺的真实物理现象。
    *   不需要繁琐的伪标签提取和二分图匹配。通过将这 64 个模态渲染回连续频谱并与 GT 计算 Loss，梯度会自动引导离真实波峰最近的预测模态靠拢（隐式匹配）。

#### 3.3 模态稀疏惩罚 (Modal Sparsity Penalty)
无论是 3.1 还是 3.2 模式，为了解决**模态数量不定**的问题，我们在总 Loss 中引入了对 `prob` 的 L1 稀疏约束：
```python
mode_loss = prob.mean()
total_loss = total_loss + mode_loss * self.sparse_penalty_weight
```
**物理意义**：该正则化项会强迫网络“偷懒”，用尽可能少的模态去拟合目标声音。对于多余的、未匹配到真实频率的模态，其 `prob` 会被梯度强力压制到 0，从而自然“关闭”，完美实现了模态数量的自适应。

## 3. 损失函数设计 (Loss Functions)

在 `compute_loss_terms` 中，系统采用多目标的联合监督：

1.  **基础回归损失 (Smooth L1 Loss)**：
    *   直接计算预测频谱与 GT 频谱的像素级绝对误差，对离群点（异常波峰）更鲁棒。
2.  **能量分布损失 (EMD 替代方案)**：
    *   将频谱视为一种概率分布，计算预测频谱累积分布函数（CDF）与真实 CDF 之间的 L1 距离。这种做法极大地缓解了频率位置稍有偏移导致的“双重惩罚”问题，引导波峰在频率轴上平滑滑动。
3.  **能量守恒约束 (Energy Conservation)**：
    *   计算两者的总能量之差（Sum L1 Loss）。这能有效打破模型在训练初期的“全零坍塌（输出全是 0 导致 Smooth L1 也很小）”陷阱。
4.  **模态稀疏损失 (Modal Sparsity Loss)**：
    *   仅在 `"modal_anchor"` 模式下生效。惩罚 `prob` 的均值，控制模态的稀疏度。通过 `cfg.SPARSE_PENALTY_WEIGHT` 调节。

---

## 4. 配置指南与总结

要使用最新且最符合敲击声物理规律的方案，建议在配置文件中进行如下设置：

```python
# 激活模态预测模式
cfg.PREDICTION_MODE = "modal_anchor"

# 关闭固定分箱，允许模态在全局频率轴自由游走聚集
cfg.USE_MODAL_BINS = False

# 调节模态稀疏度：
# 值越大，模型保留的模态越少（倾向于只保留主共振峰）；
# 值越小，模型保留的模态越多（能拟合更多微弱的谐波细节）。
cfg.SPARSE_PENALTY_WEIGHT = 0.01
```
通过这种**“可微渲染 + 隐式匹配 + 稀疏惩罚”**的范式，模型能够以极高的参数效率，无监督地从连续音频中分解出具有明确物理意义的离散模态参数。
